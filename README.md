нижеследующие действия описаны для Linux систем

## как запустить проект
1. клонируйте git репозиторий  и перейдите в него: `git clone https://github.com/fedoranvar/EM-zadanie && cd EM-zadanie`
2. убедитесь что у вас установлен `docker` и пользователь от которого вы запускаете находиться в группе `docker`
3. запустите проект командой: `docker compose up --build -d`

## как проверить результат
1. для проверки успешного старта проекта, запустите команду: `docker compose ps`
2. если сервисы успешно запустились и `backend` сервис `healthy`, то в терминале запустите команду `curl http://localhost/` или откройте в браузере адрес `http://localhost/`

## принцип работы
```
┌─────────────────┐  (1) ┌─────────────────┐ (2) ┌─────────────────┐
│                 ┼──────►                 ┼─────►                 │
│     клиент      │      │     nginx       │     │     backend     │
│                 ◄──────┼                 ◄─────┼                 │
└─────────────────┘  (4) └─────────────────┘ (3) └─────────────────┘
```
docker поднимает два сервиса `backend` и `nginx` в одной докер сети. `nginx` занимает 80й порт хоста

1. Клиент отправляет запрос на `http://localhost`
2. Nginx проксирует запрос на backend:8000
3. Backend отвечает Nginx текстом 
4. Nginx проксирует ответ клиенту

* Имена контейнеров задаются через файл `.env`. Если не заданы, то используются значения по-умолчанию: `backend` и `nginx`.
